## Что меняем

Сейчас лендинг — это длинная карточная сетка с 14 уроками. Переделываем в **scroll-driven сторителлинг**, как на Jeton:

- На экране в каждый момент — **одно сообщение, одна мысль, один логотип, одна Stories-картинка**.
- Скроллишь — слова, цвета, логотипы и картинки **меняются плавно** (parallax, fade, sticky, pin).
- Каждый блок курса = отдельная глава истории, рассказанная от лица **результата ученика** ("ты сможешь…", "у тебя будет…"), а не от лица темы урока.
- Везде встроены **логотипы нейросетей** (ChatGPT, Claude, Gemini, Lovable, HeyGen, Midjourney, Suno, Runway, Fusion AI и т.д.) и **слоты-плейсхолдеры под Stories 9:16**, куда ты потом вставишь свои видео/картинки.

Старая программа из 14 уроков остаётся в data, но визуально подаётся через сторителлинг, а не через сетку карточек.

## Ключевая механика — "глава блока"

Каждая глава курса (1, 2, 3) строится одинаково и занимает ~2 экрана высоты с `position: sticky`:

```text
┌──────────────────────────────────────────────┐
│  [01]  БЛОК 1 · МОЗГ ПРОКАЧАН                │
│                                              │
│  Левая колонка (sticky):     Правая колонка: │
│   • Большой заголовок-       ┌──────────┐    │
│     результат                │  STORY   │    │
│   • Подзаголовок             │  9 : 16  │    │
│   • Список "ты сможешь…"     │ slot #1  │    │
│   • Логотипы ИИ-инструментов └──────────┘    │
│     этой главы (ChatGPT,     ┌──────────┐    │
│     Claude, Fusion AI)       │  STORY   │    │
│                              │ slot #2  │    │
│                              └──────────┘    │
└──────────────────────────────────────────────┘
```

При скролле:
- Заголовок и список слева **меняются по мере прохождения секции** (3 шага результата на блок).
- Stories справа **сменяют друг друга** с fade/slide.
- На фоне меняется акцентный цвет/орб.

## Логотипы нейросетей

- Качаем SVG/PNG-логотипы популярных AI-сервисов в `src/assets/ai-logos/` (chatgpt, claude, gemini, midjourney, lovable, heygen, suno, runway, elevenlabs, cursor, codex, fusion).
- Источник: simpleicons.org / официальные пресс-киты. Если не находим — рисуем монохромные wordmark-плашки в стиле дизайн-системы.
- Используем 2 паттерна:
  1. **Inline-чипы** в тексте главы ("работаешь в `[ChatGPT]` `[Claude]` `[Fusion AI]`").
  2. **Бесконечная горизонтальная лента** (marquee) между главами как разделитель — медленно ползёт.

## Stories-слоты (9:16) под твой контент

Делаем компонент `<StorySlot ratio="9:16" label="Замени на свой рилс / картинку" src?: string />`:
- По умолчанию показывает **красивый плейсхолдер** с пунктирной рамкой, иконкой play и подписью "Story slot — вставь видео или картинку".
- Поддерживает `<video autoplay muted loop playsinline>` и `<img>`.
- На мобайле сторисы становятся горизонтальной свайп-каруселью.
- Все слоты вынесены в один конфиг `src/data/stories.ts`, чтобы ты быстро подменял URL-ы.

## Анимации (как у Jeton)

Используем уже подключённый `framer-motion` + `useScroll`/`useTransform` + sticky-секции:
- **Pinned scroll** для каждой главы (sticky-контейнер высотой `200vh`, внутренний контент `100vh`).
- **Параллакс** на Stories (медленнее основного скролла).
- **Текстовые морфинги**: фразы манифеста сменяются по `scrollYProgress` сегментами.
- **Marquee-ленты** логотипов AI.
- **Цветовой переход фона** между главами (брендовый градиент → глубокий фиолет → почти чёрный).
- Уважаем `prefers-reduced-motion` — отключаем тяжёлые эффекты.

## Что технически сделаем

1. Скопируем твои логотипы Fusion в `src/assets/brand/` и подменим в `Header`/`Footer`/`Author`.
2. Скачаем/добавим SVG-логотипы AI-сервисов в `src/assets/ai-logos/`.
3. Новые компоненты:
   - `StorySlot.tsx` (плейсхолдер 9:16 с поддержкой video/img).
   - `LogoMarquee.tsx` (бесконечная лента).
   - `LogoChip.tsx` (inline-чип логотипа в тексте).
   - `StickyChapter.tsx` (универсальная sticky-глава с двумя колонками).
   - `ManifestoScroll.tsx` (sticky-секция со сменой фраз по скроллу).
4. Перепишем `routes/index.tsx` под новую последовательность секций.
5. Обновим `data/lessons.ts` → добавим `chapters` (3 главы с outcome-копирайтом и привязкой логотипов).
6. Добавим `data/stories.ts` (массив слотов с алиасами).
7. Удалим/упростим старые `ProgramBlock`, `ForWhom`, `Practice` под новую подачу — без потери контента 14 уроков (они уйдут в свёрнутый аккордеон "Полная программа" внизу главы, чтобы детали остались для тех, кто хочет).
8. Проверим в превью на десктопе и мобайле, прогоним по acceptance criteria.
